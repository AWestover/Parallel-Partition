Parallel Partition Visualization

What is this?
As part of MIT PRIMES, I worked on a Computer Science research project with the help of my mentor Bill. We considered the problem of creating a fast algorithm with good theoretical guarantees for the "Parallel Partition Problem". This website contains a visualization of two parallel partition algorithms: the Strided Algorithm, and the Smoothed Striding Algorithm (the algorithm we created).

For more information, see the poster below, see our implementation of our algorithms on GitHub and see see our paper on ArXiv

In this visualization Rectangle heights correspond to the values in the array A that we are partitioning. Below the array A are multiple subarrays, which correspond to the U_i / P_i of the Smoothed Striding / Strided Algorithms. In the visualization a rectangle is colored red if it is currently being processed, black if it was processed and did not need to be swapped, yellow if it was processed and did need to be swapped, and white if it has not yet been processed. The fact that the different levels tend to end up at about the same index is evidence that algorithms.

The visualization allows you to watch the algorithms execute on different inputs. It is especially interesting to see how the randomization in the Smoothed Striding algorithm makes it so that the "adversarial" input, which is constructed to be adversarial for the Strided algorithm, is treated just like any other case, whereas the Strided Algorithm performs very poorly on the adversarial input.
